## deckModes
Enum object which contains the different modes for the deck to be in.
It can be used for with the `setMode()` function from the [useNavigation](/core-hooks#usenavigation) hook.

```js
const deckModes = {
  NORMAL:     0,
  PRESENTER:  1,
  OVERVIEW:   2,
  GRID:       3,
  PRINT:      4,
}
```

## baseTheme
Base theme object that is always being merged into the given theme, to provide sane defaults for some things.
You can overwrite any of its properties by providing your own values in your theme.

```js
const baseTheme = {
  colors: {
    text: '#000',
    background: '#FFF'
  },
  fonts: {
    body: 'system-ui, sans-serif',
  },
  fontWeights: {
    body: 400
  },
  lineHeights: {
    body: 1.5
  },
  styles: {
    root: {
      fontFamily: 'system-ui, sans-serif'
    },
    img: {
      maxWidth: '100%',
      maxHeight: '100%',
      objectFit: 'contain'
    },
    Slide: {
      fontFamily: 'body',
      fontSize: '1.5rem'
    }
  }
};
```

## mergeThemes
This function can be used to merge different theme objects together, overwriting with the latest theme given.

### Arguments
<Arguments>
{{
  name: '...themes',
  type: 'Object',
  desc: 'The theme objects you want to merge together.'
}}
</Arguments>

### Returns
<Arguments input={false}>
{{
  name: 'combinedTheme',
  type: 'Object',
  desc: 'New Theme-UI theme object, which is the result of merging all passed themes together.'
}}
</Arguments>

### Examples
```js
import {mergeThemes} from '@mdxp/core'

const combinedTheme = mergeThemes(theme1, theme2, theme3)
```

## MDXPTypes
Some components hold a special meaning for the MDXP code and will be handled specially.  
This is the enum object which contains the different types of special MDXP components.

```js
const MDXPTypes = {
  NONE: 0,
  LAYOUT: 1,
};
```

## setMDXPLayoutType
This is a higher-order component, which turns your component into a MDXP layout component.

### Arguments
<Arguments>
{{
  name: 'component',
  type: 'elementType',
  desc: 'Component of which you would like to be a layout component.'
}}
</Arguments>

### Returns
<Arguments input={false}>
{{
  name: 'component',
  type: 'elementType',
  desc: 'MDXP Layout component of your component'
}}
</Arguments>

### Examples
```jsx
import {setMDXPLayoutType} from '@mdxp/core'

const TomatoLayout = ({children, ...props}) => (
  <div
    style={{
      width: '100%',
      height: '100%',
      overflow: 'hidden',
      backgroundColor: 'tomato',
      color: 'white'
    }}
  >
    {children}
  </div>
)

export default setMDXPLayoutType(TomatoLayout);
```

## setMDXPGroupType
This is a higher-order component, which turns your component into a MDXP group component.

### Arguments
<Arguments>
{{
  name: 'component',
  type: 'elementType',
  desc: 'Component of which you would like to be a group component.'
}}
</Arguments>

### Returns
<Arguments input={false}>
{{
  name: 'component',
  type: 'elementType',
  desc: 'MDXP Group component of your component'
}}
</Arguments>

### Examples
```jsx
import {setMDXPGroupType} from '@mdxp/core'
import {ThemeProvider} from 'theme-ui'

const ThemeGroup = ({theme, children}) => (
  <ThemeProvider theme={theme}>
    {children}
  </ThemeProvider>
);

export default setMDXPGroupType(ThemeGroup);
```
